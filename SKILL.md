# 日记助手 Diary Assistant

通过微信接收碎片化生活动态，自动按时间戳收录，每日清晨汇总为第一人称日记，按年/月分目录存档。

## 核心功能

- 📩 **消息收录** — 每条消息附带发送时间戳
- 🧠 **时间优先级** — 消息内容含时间则优先使用（支持事后补记）
- 🌙 **自动汇总** — 每天清晨 06:00 整理成第一人称日记
- 🗣️ **手动触发** — 说「收工」「睡了」即刻汇总
- 📂 **按年月归档** — `YYYY/MM/YYYY-MM-DD.md`
- ✍️ **原汁原味** — 脏话粗口口语照实记录，不净化

## 初次启动 — 交互式设置

**第一次调用时**，我会通过对话问你几个问题，帮你完成全部配置：

```
日记助手启动中...

1. 怎么称呼你？（例如：老头、阿明）
   > ________

2. 日记存在哪个文件夹？（直接回车默认 D:\03\日记\）
   > ________

3. 你的时区？（直接回车默认 UTC+8 北京时间）
   > ________

4. 用哪个聊天软件收发消息？
   [微信 / Telegram / 其他]
   > ________
```

回答完毕后自动完成以下操作：

- ✅ 创建暂存目录 `raw/`（在配置文件区，用户无需关心）
- ✅ 在用户指定的路径创建按年/月归档目录
- ✅ 生成 `SOUL.md`（你的专属身份设定）
- ✅ 生成 `USER.md`（填入你的昵称、输出路径、时区）
- ✅ 生成 `memory/FACT.md`（完整工作流规则）
- ✅ 创建消息通道或指导你扫码连接
- ✅ 注册每天 06:00 的自动汇总定时任务

### ⚠️ 关于路径分离

配置时输入的路径**只放成品日记**，配置文件和暂存数据放在 Agent 工作区，互不干扰：

```
📁 配置文件区（工作目录）    📁 日记输出区（你指定的路径）
├── SOUL.md                  └── YYYY/
├── USER.md                       └── MM/
├── memory/                           └── YYYY-MM-DD.md  ← 只有这个
└── raw/
```

## 后续修改规则

输入以下命令即可进入修改模式：

| 命令 | 作用 |
|------|------|
| `改设置` | 重新走一遍设置流程 |
| `改路径` | 只修改日记存储路径 |
| `改称呼` | 只修改对你的称呼 |
| `改时区` | 只修改时区 |
| `改通道` | 切换消息通道（微信/Telegram/QQ/飞书等） |
| `改规则` | 增删改工作流规则 |
| `加规则：...` | 直接追加一条新规则 |
| `删除规则：...` | 删除指定规则 |
| `查看规则` | 查看当前所有生效规则 |

## 文件结构

```
├── SOUL.md              # 助手身份设定
├── USER.md              # 用户偏好信息
├── memory/
│   └── FACT.md          # 工作流持久知识
├── raw/
│   └── YYYY-MM-DD.md    # 白天暂存记录
├── YYYY/
│   └── MM/
│       └── YYYY-MM-DD.md # 成品日记
```

## 工作流

```
用户发消息 → 抓取发送时间戳
  ├─ 内容含时间？→ 优先用内容里的时间（事后补记）
  └─ 否则用发送时刻
↓
追加到 raw/YYYY-MM-DD.md
↓ 回复「已记下 ✅」
↓
[06:00 定时 / 或收到"收工/睡了"]
↓
读取 raw/...md → 按时间线写成第一人称日记
↓
写入 YYYY/MM/YYYY-MM-DD.md
↓ 删除暂存文件 → 发送通知
```

## 核心原则

1. **时间即骨架** — 每条记录附带准确时间戳
2. **真实至上** — 不编造、不净化、不删减
3. **日日不辍** — 无论多少，每日必汇总
4. **口吻如一** — 第一人称「我」，像用户自己写的
5. **时间优先级** — 消息内容里的时间 > 发送时刻
